Describes how to design exploratory design games and discusses some concrete games and their effects. 
## Types of exploratory design games
### Games to Conceptualize Designing
Create configurations using game pieces on a game board. Neither refer to any real life artifact but is abstracted/stylized to eliminate the functional knowledge designers bring with them. 
Focus is to explore interactions between players / learn about the concepts game players hold?
Neither game includes users. 
#### Exchange Perspective Games
Games to explore imagination. Use elements of change. Based on wish to get out of habits and experience new. 
#### Negotiation / Workflow oriented Games
Creating a common understanding of the work context to design for. 
Participants played themselves. Objective was to engage workers in change process - create common language, discuss existing reality, investigate future visions and make requirement specifications. 

e.g. Organisational Kit Game. Designers made field analysis beforehand, put findings on cards. Cards are placed on game board, their meaning discussed and thus a common understanding created of the current organisation. 
#### Scenario oriented Design Game
_Forum Theatre_ - a play is enacted. Audience suggest changes, after a debate the play is performed again with the incorporated changes. 

_The magic if_ - when creating a role, actress has to ask herself questions like "what if the character was in this or this situation" - how would she react -> easy to ask similar questions when focusing on user experiences or exploring use contexts.

3 example games developed in the Design Lab
#### User Game
Key frame from user video snippets & sign cards (with a word)
Create stories with the key frame cards that are labeled with a sign cards. Ordered in a crossword like manner (key frame cards belong to more than one story). 

Similar to Personas. 
Aim: To overcome difficulties in communicating users needs to designers. Get designers to understand users. Understand the people who the design is for. 

In personas users don't participate in their creation which can be a disadvantage. 
User Game keeps the user more in mind. 
#### Landscape Game
Future workshops - create visions by structured participation. Critique phase, participants critize current work practice -> clustered; fantasy phase, visions/ideas to deal with problems identified; implementation phase, return to present, identify obstacles and plan activities to realize aspects of vision. 

Workplace making games - walk through, with participants identifying parts on the fotos/arranging their workplace with card board pieces

Landscape game - create context for personas created in user game

- important that the games engage the participants and are played in an informal atmosphere
- game pieces functions as boundary objects - shared objects, which at the same time allow for different interpretations

### Discussion
Game pieces become inherent part of the language and herby the argumentation. Together with formulation of rules on how to play, this contributes to levelling of stakeholders with different views leading to a more constructive dialogue. 

When involving people with various backgrounds, competencies and professional languages, game materials need to function as boundary objects. 

Exploratory design games are engaging and fun for people to take part in. Game playing creates an informal atmosphere, which is the most productive in creative work.
